# Streamlit 기반 CIAM 공격 분석 대시보드 구축 작업지시서

본 문서는 `attack_ciam.db` 내 회원 가입 신청 데이터를 시각적으로 모니터링하고, 특정 키워드를 기반으로 공격 그룹의 행동 패턴을 교차 분석하는 **Streamlit 대시보드 애플리케이션 개발을 위한 작업지시서(Work Instruction)**입니다.

---

## 1. 프로젝트 개요 및 환경

- **목적**: 보안 운영팀(SOC)이 가입 데이터 내에서 발생하는 비정상 봇 대량 가입(Fake Registration) 및 Gmail Dot Trick 남용 패턴을 신속하게 모니터링하고 분석할 수 있는 웹 기반 대시보드 구축
- **기술 스택**:
  - **웹 프레임워크**: `Streamlit`
  - **시각화 엔진**: `Plotly` (동적 인터랙티브 차트 구현)
  - **데이터베이스**: `SQLite3` (`attack_ciam.db`)
  - **데이터 가공**: `Pandas`, `NumPy`

---

## 2. 대시보드 레이아웃 및 탭(Menu) 구조

대시보드는 크게 **좌측 사이드바 필터 패널**과 **우측 메인 분석 영역**으로 분할하여 직관적으로 설계합니다.

### 2.1 좌측 사이드바 (Sidebar Controls)
- **데이터베이스 연결 설정**: `attack_ciam.db` 경로 확인 정보창
- **키워드 검색기 (Keyword Filter)**: 
  - 검색 입력란 제공 (이메일, 회사명, 회사코드 등 통합 검색)
  - 검색 대상 컬럼 선택 옵션 (전체, requestor_email, requestor_company_name, requestor_company_code)
- **채널 멀티 셀렉터 (Channel Multi-select)**: 가입 경로별 데이터 필터링

### 2.2 우측 메인 대화 영역 (Main Dashboard)
상단에 전체 요약 지표 카드(Metric Card) 4종을 배치하고, 하단은 2개의 탭 메뉴로 구성합니다.

* **상단 요약 지표 (KPI Metrics)**:
  - **총 가입 시도 건수**
  - **Gmail 계정 비중 (%)**
  - **초고속 자동 승인율 (%)** (신청 후 1초 이내 승인)
  - **결측 계정 수** (미승인 대기 계정)

* **Tab 1: 기초 EDA (탐색적 데이터 분석)**:
  - 전체 수집 데이터에 대한 기본 형태와 시각화 자료를 제공
  - **5가지 그래프** 및 **5가지 기술 통계 표**를 순차적으로 매핑
* **Tab 2: 키워드 기반 공격 패턴 비교 (Pattern Analytics)**:
  - 사용자가 입력한 검색어 키워드 매칭 그룹 vs 전체(또는 비매칭) 그룹의 핵심 지표 교차 비교

## 3. 세부 구현 명세

### 3.1 구현할 동적 그래프 5가지 (Plotly 시각화 스펙)

모든 시각화는 마우스 오버 시 상세 값이 노출되고, 동적으로 줌/패닝이 가능한 `Plotly` 차트로 개발합니다.

1. **Plotly 시각 선그래프 (Line Chart)**: *시간대별 가입 신청 분포*
   - **X축**: 24시간대 (0시 ~ 23시)
   - **Y축**: 가입 신청 수 (건)
   - **스펙**: 시간 흐름에 따른 기계적 트래픽 유입 곡선 표현
2. **Plotly 도넛 차트 (Donut Chart)**: *가입 유입 경로(Channel)별 점유 비율*
   - **데이터**: `channel` 컬럼 빈도 비중
   - **스펙**: 가입 유입구 쏠림을 시각적으로 인지할 수 있도록 파이/도넛 형태로 구현
3. **Plotly 가로 막대 그래프 (Horizontal Bar Chart)**: *이메일 도메인 Top 10 분포*
   - **Y축**: 이메일 도메인 문자열
   - **X축**: 계정 수 (건)
   - **스펙**: 스팸/일회용 임시 메일 도메인이 어느 정도 점유하는지 정렬하여 출력
4. **Plotly 세로 막대 그래프 (Vertical Bar Chart)**: *이메일 ID 내 온점(dot) 개수 분포*
   - **X축**: 이메일 ID에 포함된 온점 개수 (0개, 1개, 2개 ...)
   - **Y축**: 계정 수 (건)
   - **스펙**: Gmail Dot Trick 어뷰징 흔적(마침표 다수 삽입)을 직관적으로 증명
5. **Plotly 가로 막대 그래프 (Horizontal Bar Chart)**: *가입 신청 건수 Top 15 회사 코드*
   - **Y축**: 회사 코드 (회사명 결합 포맷)
   - **X축**: 가입 신청 수 (건)
   - **스펙**: 대량 가짜 가입이 집중된 타겟 도메인/회사 코드를 모니터링

### 3.2 구현할 통계 표 5가지 (Pandas & Streamlit 테이블 스펙)

모든 그래프 하단에는 해당 차트의 수치를 엄밀하게 증명하는 테이블을 매칭하여 배치합니다.

1. **시간대 x 채널 가입 신청 교차 피봇 테이블 (Crosstab Table)**
   - **행**: 시간대 (0 ~ 23)
   - **열**: 상위 주요 유입 채널 4종 및 기타 채널 그룹
   - **값**: 가입 신청 수
   - **위치**: *시간대별 유입량 선그래프* 하단 배치
2. **유입 경로(Channel)별 빈도 분포 및 누적 비율 통계 표**
   - **컬럼**: 채널명, 가입 시도 건수, 비율(%), 누적 비율(%)
   - **위치**: *채널 비율 도넛 차트* 하단 배치
3. **이메일 도메인 Top 10 빈도표**
   - **컬럼**: 순위, 도메인 주소, 계정 수, 비율(%), 도메인 신뢰 수준 (일회용 메일 분류 포함)
   - **위치**: *이메일 도메인 분포 차트* 하단 배치
4. **온점(dot) 개수 분포 기술 통계 요약 및 빈도 분포표**
   - **상단**: 온점 수 요약 통계량 (전체 표본 수, 평균, 표준편차, 중앙값, 최솟값, 최댓값)
   - **하단**: 온점 개수별 계정 건수 및 백분율 매칭 테이블
   - **위치**: *온점 개수 분포 막대그래프* 하단 배치
5. **Top 15 공격 의심 회사 코드 통계 표 (최빈 가입 시간대 결합)**
   - **컬럼**: 회사 코드, 회사명, 가입 건수, 비율(%), 트래픽 피크 시간대(최빈 시간)
   - **위치**: *회사 코드 분포 막대그래프* 하단 배치

## 4. 키워드 검색 기반 공격 패턴 비교 (Tab 2 상세 스펙)

사용자가 입력한 검색어를 기준으로 타겟 공격자 그룹과 일반 그룹의 패턴 차이를 한 눈에 비교하는 분석 기능입니다.

### 4.1 키워드 필터링 로직
- 검색어가 입력되면 다음 조건으로 데이터를 슬라이싱합니다:
  ```python
  # 키워드 검색 필터링 예시 코드
  keyword_lower = search_keyword.strip().lower()
  matched_df = df[
      df['requestor_email'].str.lower().str.contains(keyword_lower) |
      df['requestor_company_name'].str.lower().str.contains(keyword_lower) |
      df['requestor_company_code'].str.lower().str.contains(keyword_lower)
  ]
  non_matched_df = df[~df.index.isin(matched_df.index)]
  ```

### 4.2 교차 비교 지표 카드 (KPI Comparison)
- 검색 매칭 데이터 그룹과 비매칭(혹은 전체) 그룹의 3대 이상 징후 비율을 나란히 배치해 시각화합니다.
  - **평균 온점 개수 비교** (예: 검색 그룹 8.5개 vs 전체 평균 3.9개)
  - **초고속 자동 승인율 비교** (예: 검색 그룹 100% vs 전체 97.7%)
  - **특정 채널 유입 집중도 비교** (예: 검색 그룹 `edo_attack` 비율)

### 4.3 비교 시각화 차트
- Plotly의 Grouped Bar Chart(그룹화된 막대 차트)를 이용하여 검색 키워드 매칭 그룹과 전체 그룹의 시간대별 가입 신청 트래픽 곡선 또는 채널 유입 비중 차이를 대조 출력합니다.

## 5. 단계별 개발 가이드 (개발자 지침)

```bash
pip install streamlit plotly pandas numpy
```

### Step 2. 데이터베이스 연동부 개발
- `sqlite3`를 통해 `attack_ciam.db`에 안전하게 연결하고 데이터를 판다스 데이터프레임으로 변환하는 캐싱 기법(`@st.cache_data`) 적용:
```python
import streamlit as st
import sqlite3
import pandas as pd

@st.cache_data
def load_data():
    conn = sqlite3.connect("attack_ciam.db")
    df = pd.read_sql_query("SELECT * FROM attack_account", conn)
    conn.close()
    return df
```

### Step 3. 메인 UI 및 탭 기능 구현
- `st.sidebar`를 구성하고 메인 영역에 `st.tabs(["기초 EDA", "키워드 패턴 비교"])`를 호출하여 탭 전환 레이아웃 설계

### Step 5. 키워드 비교 로직 및 결과 리포트 구현
